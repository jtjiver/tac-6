### Key Innovation: Task Tool for Sub-Agents

Instead of manually running each slash command, we use the **Task tool** to delegate to specialized sub-agents:

```markdown

You run: /classify_issue

Task tool spawns: "Generate branch name for {type} issue {number}"
Task tool spawns: "Create implementation plan for {issue}"
...
```

**Benefits:**
- ‚úÖ Fully automated - just provide issue number
- ‚úÖ Sub-agents run in parallel when possible
- ‚úÖ Still $0 cost (same Claude Code session)
- ‚úÖ More robust error handling
- ‚úÖ Better progress tracking

### When to Use Task Tool vs Direct Slash Commands

**Use Task Tool (Sub-Agent) When:**
- ‚úÖ Task requires research/analysis (like fetching issue)
- ‚úÖ Task generates content independently
- ‚úÖ Task needs error handling/retries
- ‚úÖ You want parallel execution
- ‚úÖ Task is complex and benefits from focused attention

**Use Direct Slash Command When:**
- ‚úÖ Task is classification or generation (slash commands exist)
- ‚úÖ Task creates agent artifacts
- ‚úÖ Task is deterministic
- ‚úÖ You want immediate inline execution
